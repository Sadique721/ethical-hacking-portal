2. SYSTEM WORKFLOW & REQUEST CYCLES
------------------------------------------------------------------------
- Request Routing:
  1. Client sends a request (e.g. GET to http://127.0.0.1:8000/).
  2. `myfirstpro/urls.py` processes the request, matches it, and hands it off to `MSA/urls.py`.
  3. `MSA/urls.py` resolves the path and invokes the appropriate function in `MSA/views.py`.

- Authentication & Profiles:
  - Registration: `views.registers` handles user creation by validating fields in `forms.py`. 
  - Signals: When a new User is created, a `post_save` signal (in `models.py`) automatically generates a corresponding `Profile` instance in the database.
  - Login/Logout: standard Django session-based login (`views.user_login`) and logout.
  - Protected views: Views like `profile`, `edit_profile`, and `dashboard` use the `@login_required` decorator to enforce authorization.

- Static File Serving:
  - Development: Django automatically retrieves files from directories listed in `STATICFILES_DIRS`.
  - Production (Render): WhiteNoise middleware (`whitenoise.middleware.WhiteNoiseMiddleware`) intercepts static requests and serves files directly from the compiled `STATIC_ROOT` folder (`staticfiles/`), providing compression and caching.

4. HOW TO RUN LOCALLY
------------------------------------------------------------------------
1. Install Python 3 (3.8 - 3.13 recommended).
2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
3. Run migrations:
   ```bash
   python manage.py makemigrations MSA
   python manage.py migrate
   ```
4. Run server:
   ```bash
   python manage.py runserver
   ```
5. Visit http://127.0.0.1:8000/ in your browser.


------------------------------------------------------------------------
5. RENDER.COM DEPLOYMENT PROCESS
------------------------------------------------------------------------
This project is configured to deploy directly to Render.com out-of-the-box using the following configurations:

1. Create a Web Service:
   - Log into Render.com and connect your GitHub repository.

2. Set Environment & Start Commands:
   - Environment: Python
   - Build Command:
     ```bash
     python -m pip install -r requirements.txt && python manage.py collectstatic --noinput && python manage.py migrate
     ```
   - Start Command:
     ```bash
     gunicorn myfirstpro.wsgi:application --bind 0.0.0.0:$PORT
     ```

3. Environment Variables (optional but recommended for production):
   - Set `DEBUG` to `False` (for security).
   - Set `SECRET_KEY` to a custom production string.
   - Set `ALLOWED_HOSTS` to your Render service sub-domain (e.g. `your-app-name.onrender.com`).
